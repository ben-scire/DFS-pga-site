== WEEKLY WORKFLOW ==

One command pulls scores from Firestore and recomputes standings:

  cd league-scoring
  python3 update_league_standings.py --contest-id week-3-players

This will:
  1. Connect to Firestore using GOOGLE_APPLICATION_CREDENTIALS from .env
  2. Read test_lineups/{contestId}/entries (each entry's lineup)
  3. Read test_scores/{contestId}/golfers (fantasy points per golfer)
  4. Sum fantasy points for each entry's lineup
  5. Write weekly-scores/{contestId}.json
  6. Warn if any entry-users.json members are missing from Firestore
  7. Recompute season-standings.json from all weekly-scores files

If anyone is missing from Firestore (the script will warn you), manually
add them to the weekly JSON file and re-run without --contest-id:

  python3 update_league_standings.py

== PREREQUISITES (one-time setup) ==

  - Firebase service account JSON in the project root (e.g. studio-*.json)
  - .env file in the project root with:
      GOOGLE_APPLICATION_CREDENTIALS=./studio-5115982885-551c8-b97cd22c73e7.json
  - Python packages: pip install google-cloud-firestore google-auth

== OTHER MODES ==

Recompute standings only (no Firestore, uses existing weekly-scores files):
  python3 update_league_standings.py

Pull Ben's DFS API scores for comparison:
  python3 update_league_standings.py --ben --contest-id week-2-arnold-palmer
  (Requires DFS API credentials in .env)
