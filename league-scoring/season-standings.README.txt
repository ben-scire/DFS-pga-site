== WEEKLY WORKFLOW ==

One command pulls scores from Firestore and recomputes the standings outputs:

  cd league-scoring
  python3 update_league_standings.py --contest-id week-3-players

This will:
  1. Connect to Firestore using GOOGLE_APPLICATION_CREDENTIALS or FIREBASE_SERVICE_ACCOUNT_JSON
  2. Read test_lineups/{contestId}/entries
  3. Read test_scores/{contestId}/golfers
  4. Prefer officialWeeklyFantasyPoints when present, otherwise sum golfer fantasy points
  5. Write weekly-scores/{contestId}.json
  6. Warn if known entries are missing from Firestore
  7. Recompute standings-template.json and season-standings.json

If anyone is missing from Firestore, add them manually to the weekly JSON file and
rerun the script without --contest-id:

  python3 update_league_standings.py

== PREREQUISITES ==

  - Firebase service account JSON in the project root, or
  - FIREBASE_SERVICE_ACCOUNT_JSON set in .env / environment
  - Optional GOOGLE_CLOUD_PROJECT when using inline service account JSON
  - Python packages:
      python3 -m pip install google-cloud-firestore google-auth

== OTHER MODES ==

Recompute standings only (no network, uses existing weekly-scores files):
  python3 update_league_standings.py

Pull Ben's DFS API scores for comparison:
  python3 update_league_standings.py --ben --contest-id week-2-arnold-palmer
